# Comando: Handoff

Crie ou atualize o arquivo `docs/HANDOFF.md` no diretório atual.

## Instruções

1. Se o arquivo já existe, leia-o primeiro e atualize com o progresso atual
2. Se não existe, crie usando o template padrão

## O documento DEVE conter

### 1. Objetivo Atual
- O que estamos tentando fazer nesta sessão
- Em 1-2 frases claras

### 2. O Que Foi Feito
- Lista de tarefas completadas nesta sessão
- Resumo do que foi feito em sessões anteriores

### 3. O Que Falta
- Pendências imediatas (próxima sessão)
- Pendências de curto prazo
- Backlog futuro

### 4. Decisões Tomadas
- Decisões arquiteturais e suas justificativas
- Decisões de regras de negócio

### 5. Problemas Encontrados
- Problemas resolvidos e suas soluções
- Problemas pendentes e status

### 6. Arquivos Modificados
- Lista de arquivos criados
- Lista de arquivos modificados (com descrição do que mudou)
- Lista de arquivos excluídos

### 7. Estado Atual do Sistema
- O que está funcionando
- O que está parcialmente funcionando
- O que não foi implementado ainda

### 8. Próximos Passos
- Em ordem de prioridade
- Com detalhes do que fazer
- Indicando arquivos envolvidos

### 9. Contexto Crítico
- Variáveis de ambiente necessárias
- Comandos para retomar o trabalho
- Informações importantes que não podem ser perdidas

## Regras

- Seja conciso mas completo
- O próximo agente só terá este documento
- Inclua tudo que seria necessário para outra pessoa continuar
- Atualize a data no topo do documento
- Use checkboxes para tarefas (- [ ] ou - [x])

## Quando Usar

- Antes de `/clear` para preservar contexto
- Ao final de uma sessão de trabalho
- Quando contexto passar de 60%
- Ao trocar de tarefa principal
